# 발표 큐카드 — 우리동네 공구 (15~20분)

> 스크립트 **그대로 읽지 말고** 키워드만 참고하세요.

---

## 1. 오프닝 (1분) — 슬라이드 1~2

- 인사, 팀명·발표자 소개
- **한 줄:** "동네 이웃과 함께 생필품을 싸게 사는 공동구매 앱입니다."

---

## 2. 문제 · 타겟 (2분) — 슬라이드 3~4

**문제**
- 쿠팡·마켓컬리는 **1인 구매** → 대량 할인 혜택 적음
- 카카오톡 공구는 **신뢰·환불·위치** 불명확

**타겟**
- **참여자:** 20~40대, 생필품 반복 구매, 동네 1~2km
- **팀장:** 이웃에게 도움 + 소액 수수료, 부담 낮은 역할

---

## 3. 컨셉 · 근거 (2분) — 슬라이드 5~6

- **컨셉:** 플랫폼이 품목·가격 제공 + 동네 수령 공구
- **근거:** 10~50명 묶음 → 단가↓, 공구페이로 먹튀 방지
- **차별점:** 위치 기반 탐색, 등급제 팀장, 예외 플로우까지 설계

---

## 4. 컨셉 도출 (1.5분) — 슬라이드 7

- 수업: IA·와이어프레임 → **엣지케이스 워크숍** (edge_case.md)
- vision.md 11개 요구 → 반복 프로토타이핑
- "와이어프레임만으로는 멀리서 참여 문제 발견 → 위치 입력 추가"

---

## 5. IA (2분) — 슬라이드 8

- 8개 IA 그룹, 38+ 화면
- 탭: 홈·탐색·공구열기·채팅·마이
- **강조:** 예외·환불을 1급 메뉴가 아닌 **플로우 분기**로 설계

---

## 6. 유저플로우 (3분) — 슬라이드 9~10

**참여자:** 탐색 → 상세 → 공구페이 → 모집 → 채팅 → 수령 → 후기  
**팀장:** 카탈로그 선택 → 위치 → 3단계 개설 → 초대  
**분기:** 모집 실패 환불, 팀장 연락 두절, 분쟁 신고

→ `/screens` 보드로 한 번에 보여주면 효과적

---

## 7. 핵심 기능 (4분) — 슬라이드 11~13

| 기능 | 한 줄 |
|------|------|
| 공구페이 | 수령 전 에스크로 |
| 플랫폼 카탈로그 | 허위 가격·품목 방지 |
| 위치·범위 | 수령 거리 + 참여/노출 이중 범위 |
| 등급제 | 팀장 동기 부여, 수수료 50→95% |
| 최소 인원 | 카드에 "최소 N명" 표시 |

**데모:** explore → detail → open (2~3분)

---

## 8. 예외 UX (1.5분) — 슬라이드 14

- edge_case 5개 카테고리 → 각각 전용 화면
- "실패 케이스를 숨기지 않고 설계에 넣었다"

---

## 9. 프로토타입 · 디자인 (2분) — 슬라이드 15~17

- 와이어프레임 → **동작 프로토타입** (React, Vercel 배포)
- Lightroom 톤 UI, 실사 상품·OSM 지도
- URL 공유

---

## 10. 마무리 (1.5분) — 슬라이드 18~19

- 배운 점: 엣지케이스가 IA를 키움
- 한계: 실제 결제·배송 API 미연동 (데모)
- Q&A
