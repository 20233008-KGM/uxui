## 해결하려는 문제

- **1인 구매** → 대량 할인 혜택 제한
- **카톡 공구** → 환불·먹튀·위치 불투명
- **팀장 부담** → 개설·참여 진입 장벽

**기회:** 동네 + 생필품 반복 구매 + 에스크로 신뢰

---

## 타겟 사용자

| 페르소나 | 니즈 |
|----------|------|
| **참여자** | 저렴한 생필품, 가까운 수령, 안전 결제 |
| **팀장** | 간편 개설, 수수료, 프라이버시 |
| **플랫폼** | 허위 등록·분쟁 최소화 |

---

## 서비스 컨셉

> 플랫폼이 품목·가격 제공 → 이웃 팀장이 동네에서 모아 수령

- **공구페이** 에스크로
- **플랫폼 카탈로그** (고정가 5종)
- **위치·범위** 기반 탐색
- **팀장 등급제** (B~P, 수수료 50~95%)

---

## IA (정보 구조)

```
진입·인증 → 메인·탐색 → 공구열기 → 결제·모집
→ 채팅·수령 → 예외·환불 → 마이페이지 → 이벤트
```

하단 탭: 홈 | 탐색 | ＋ | 채팅 | 마이  
**38+ 화면**

## 유저플로우 — 참여자

`홈 → 탐색 → 상세 → 공구페이 → 모집 → 채팅 → 수령 → 후기`

분기: 모집 실패(환불) · 수령 기한 초과 · 분쟁

---

## 유저플로우 — 팀장

`공구열기 → 카탈로그 → 위치 → 인원·일정 → 공구페이 → 개설 → 초대`

모집 **10~50명** · 최소 인원 카드 표시
