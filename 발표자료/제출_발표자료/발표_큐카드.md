# 우리동네 공구 · 발표 큐카드

> ⚠️ **스크립트 그대로 읽으면 감점.** 아래는 슬라이드별 **키워드/포인트**만. 본인 말로 풀어서 발표.
> 목표 시간 **15~20분** · 16슬라이드 → 슬라이드당 평균 **약 60~70초**.

---

### 1. 표지 (약 40초)
- 인사 + 팀명/발표자
- 한 줄 후킹: "혼자 사면 비싸고, 같이 사면 불안하다 — 그 사이를 잇는 서비스"

### 2. 목차 (약 30초)
- 6개 줄기만 빠르게: 문제·타겟 → 컨셉 도출 → 컨셉·근거 → IA → 플로우·프로토타입 → 엣지케이스

### 3. 문제 (약 70초) ★컨셉 납득의 출발점
- 1인 구매 = 대량할인 소외 / 카톡 공구 = 먹튀·환불·위치 불투명 / 팀장 부담
- 마무리 문장: "**신뢰만 해결되면** 동네 공구는 가장 자연스러운 절약"

### 4. 타겟 사용자 (약 70초)
- 절약형 참여자 · **동네 팀장(핵심)** · 플랫폼 → **양면 시장**
- 강조: 팀장 공급 동기 설계가 성패를 가른다

### 5. 인사이트 (약 60초)
- 동네 · 반복 · 신뢰 세 개가 겹치는 자리
- "문제는 가격이 아니라 신뢰"

### 6. 컨셉 도출·리서치 (약 80초) ★강의 내용 연결
- 관찰/인터뷰 → 페인포인트 → 가설 → 프로토타입 검증 → 14차 이터레이션
- 발견→해석→전환 3단으로: 신뢰 문제 → 구조로 해결 → 공구페이/카탈로그/등급제

### 7. 서비스 컨셉 (약 70초)
- 한 줄: "플랫폼이 품목·가격 제공, 이웃 팀장이 동네에서 모아 수령"
- 4기둥: 고정가 카탈로그 / 공구페이 에스크로 / 범위 매칭 / 등급제

### 8. 기능 × 근거 (약 70초)
- 기능마다 "왜"가 있다 — 표 빠르게 훑기 (먹튀·허위·원거리·공급 동기)

### 9. 팀장 등급제 (약 80초) ★차별점
- 브론즈50 → 실버65 → 골드80 → **플래티넘95%**
- 개설당 +250 마일리지 / 기준 수수료 800원 → 플래티넘이면 760원이 팀장 몫
- "거부감을 보상으로 바꾼다"

### 10. 정보 구조도 IA (약 70초)
- 8개 그룹 · 38개 화면 · 전역 하단 탭(홈·탐색·＋·채팅·마이)
- 로그인 없이 탐색까지, 결제·개설부터 인증 → 진입장벽 뒤로

### 11. 유저플로우 — 참여자 (약 60초)
- 홈→탐색→상세·지도→공구페이→모집·채팅→수령·후기 / 실패 시 자동 환불 분기

### 12. 유저플로우 — 팀장 (약 60초)
- 공구 열기→카탈로그→위치·반경→인원·일정→개설→지인 초대
- "가격 입력이 없다" 강조 (이미 협상된 단가)

### 13. 프로토타입 갤러리 (약 50초)
- 와이어프레임 → 동작형 React 프로토타입 / 라이브 데모 URL 언급

### 14. 엣지케이스 (약 70초) ★완성도 어필
- 모집 실패·연락두절·기한초과·취소·콜드스타트 → 각 전용 화면/정책
- "잘 풀릴 때가 아니라 틀어질 때를 설계"

### 15. 구현 요약 (약 50초)
- 36 페이지 · 38 화면 · 14차 이터레이션 · 4등급
- 스택: React19/TS/Vite/Tailwind4 · OSM 지도 · 실사 WebP · Vercel

### 16. 마무리 (약 30초)
- "동네에서 같이 사면, 같이 싸게" + Q&A 유도 + 데모 URL

## 예상 질문 대비
- **Q. 수익모델?** → 거래 수수료(기본 800원/인) 중 등급별 분배. 플랫폼 몫은 신뢰 인프라(에스크로·CS) 운영비.
- **Q. 콜드스타트 어떻게?** → 범위 확장 설정 + 초대 이벤트(양면 보상) + 동네 단위 밀도 집중.
- **Q. 팀장 95% 주면 플랫폼은?** → 등급은 활성 공급자 락인 장치. 거래량·리텐션·바이럴로 보전.
- **Q. 가격을 고정한 이유?** → 허위·흥정·정산 분쟁 제거. 신뢰 = 핵심 가치.
