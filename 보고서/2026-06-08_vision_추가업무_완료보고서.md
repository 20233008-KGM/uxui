## #1 공구 열기 시 위치 입력

### 원문 요청
> 공구열기 해도 위치 입력할 방법이 없어어

### 완료 여부
✅ **완료**

### 수행 내용

공구 열기 1단계(`/open`)에 **수령 위치(팀장) 필수 입력** UI를 추가했습니다.

- **현재 위치 사용** 버튼 — GPS 대신 데모용 `userHome` 좌표 적용
- **프리셋 동네 선택** — `leaderLocationPresets` 목록에서 선택
- **상세 주소 입력** — 아파트·동·호 등 보조 주소
- **OpenStreetMap 지도 미리보기** — `LocationPicker` + `MapPreview` 컴포넌트
- 위치 미선택 시 **다음 단계 버튼 비활성화** (`canProceed` 검증)

### 관련 파일

| 파일 | 역할 |
|------|------|
| `src/pages/OpenGroupBuyPage.tsx` | 위치 입력 섹션, 진행 조건 |
| `src/components/LocationUI.tsx` | `LocationPicker` — 지도·프리셋·주소 |
| `src/components/MapPreview.tsx` | OSM iframe 지도 |
| `src/data/locations.ts` | 좌표·프리셋 데이터 |
| `src/data/mapUtils.ts` | OSM embed/외부 링크 URL |

### 확인 방법
1. `/open` 접속  
2. 「수령 위치 (팀장)」에서 동네 선택 또는 「현재 위치 사용」  
3. 지도가 표시되고 「다음 단계」 활성화 확인

## #2 팀장 지인 초대 → 지인 혜택 이벤트 창

### 원문 요청
> 공구 팀장이 지인을 초대해서 지인이 초대를 받으면 뭔가 혜택이 있는 이벤트 창 열어줘.

### 완료 여부
✅ **완료**

### 수행 내용

**팀장 측 (초대 발송)**

- `/leader/invite` — 초대 코드·링크 생성, 복사·공유 UI
- 초대 링크 형식: `{origin}/invite/{CODE}` (예: `/invite/KIM2024`)

**지인 측 (초대 수신)**

- `/invite/:code` — 전용 **이벤트 팝업형 화면**
- 팀장 이름·공구명 표시
- 혜택: **1,000P + 500원 할인** (`defaultInviteReward`)
- 「혜택 받기」 → 포인트 적립 후 공구 상세 또는 회원가입으로 이동

### 혜택 수치 (`src/data/inviteEvent.ts`)

| 대상 | 혜택 |
|------|------|
| 지인 (피초대자) | 1,000P + 500원 할인 |
| 팀장 (초대자) | 300P 적립 |

### 관련 파일

| 파일 | 역할 |
|------|------|
| `src/pages/InviteFriendsPage.tsx` | 팀장 초대·링크 복사 |
| `src/pages/InviteEventPage.tsx` | 지인 수신 이벤트 창 |
| `src/data/inviteEvent.ts` | 혜택·데모 초대 코드 |
| `src/context/AppContext.tsx` | `claimInviteReward()` 상태 처리 |

### 확인 방법
1. `/leader/invite` — 링크 복사  
2. `/invite/KIM2024` 또는 `/invite/FRIEND` — 이벤트 창·혜택 수령 UI 확인

## #3 참여·노출 이중 범위 설정 (마이페이지)

### 원문 요청
> 물건을 사거나 팔때 어느범위의 지역까지의 공구를 참여할것인지 팀장은 어느 범위의 지역역까지의 참여자들에게 물건을 보여줄 것인지에 대한 범위 설정을 본인이 할수 있도록 마이페이지에서 수정필요

### 완료 여부
✅ **완료**

### 수행 내용

마이페이지 → **「동네 범위 설정」** (`/mypage/neighborhood`)에서 두 가지 범위를 **독립적으로** 설정합니다.

| 설정 | 대상 | 설명 |
|------|------|------|
| **참여 범위** | 일반 참여자 | 내 동네 기준, 이 거리 안의 공구만 탐색·참여 |
| **노출 범위** | 팀장 | 내가 연 공구를 이 거리 안 이웃에게만 노출 |

선택 옵션: **500m / 1km (권장) / 2km / 3km** (`rangeLabels`)

- 홈·탐색 필터: `participationRange` 기준 공구 목록 필터링
- 공구 열기 1단계: `openVisibilityRange` — 해당 공구만의 노출 범위 (마이페이지 기본값 상속)

### 관련 파일

| 파일 | 역할 |
|------|------|
| `src/pages/NeighborhoodSettingsPage.tsx` | 이중 범위 UI |
| `src/pages/MyPage.tsx` | 「동네 범위 설정」 메뉴 링크 |
| `src/data/notifications.ts` | 범위 라벨·설명 문구 |
| `src/context/AppContext.tsx` | `participationRange`, `leaderVisibilityRange`, `openVisibilityRange` |
| `src/pages/HomePage.tsx`, `ExplorePage.tsx` | 참여 범위 필터 적용 |

### 확인 방법
1. `/mypage` → 「동네 범위 설정」  
2. 참여 범위·노출 범위 각각 변경 후 저장  
3. `/open` — 공구별 노출 범위 그리드 확인  
4. `/` — 참여 범위에 맞는 공구만 노출 확인

## #4 팀장 등급제 · 반짝 이름 · 수수료 · 다음 등급

### 원문 요청
> 회원 등급을 만들고 등급에 따라 마이페이지에서 이름에 반짝반짝 느낌의 장식이 나타나게 해줘. … 브론즈, 실버, 골드, 플레티넘 … 플레티넘 등급을 달성하면 거래 수수료의 대부분을 팀장의 몫으로 … 마이페이지에서 본인의 등급을 확인 … 다음등급까지 얼마나 남았는지도 확인

### 완료 여부
✅ **완료**

### 수행 내용

**4단계 등급** (`src/data/leaderTier.ts`)

| 등급 | 필요 마일리지 | 팀장 수수료 비율 |
|------|--------------|-----------------|
| 브론즈 | 0 | 50% |
| 실버 | 500 | 65% |
| 골드 | 1,500 | 80% |
| 플래티넘 | 3,000 | **95%** (대부분 팀장 몫) |

**마일리지 적립:** 공구 1회 개설 시 +250 (`MILEAGE_OPEN_GROUP_BUY`)

**UI 구현**

- 마이페이지 — `SparkleName` 컴포넌트로 **등급별 반짝 CSS 애니메이션** (`index.css`의 `.tier-name-*`)
- `TierBadge`, `TierProgressCard` — 현재 등급·진행률·다음 등급까지 남은 마일리지
- `/mypage/tier` — 등급 상세, 등급별 수수료표, 적립 방법 안내
- 공구 열기 3단계 — 등급별 예상 수수료 미리보기

### 관련 파일

| 파일 | 역할 |
|------|------|
| `src/data/leaderTier.ts` | 등급·마일리지·수수료 계산 |
| `src/components/TierUI.tsx` | `SparkleName`, `TierBadge`, `TierProgressCard` |
| `src/index.css` | 등급별 shimmer/sparkle 스타일 |
| `src/pages/MyPage.tsx` | 이름 반짝·등급 배지·진행 카드 |
| `src/pages/LeaderTierPage.tsx` | 등급 상세·다음 등급 안내 |
| `src/pages/OpenGroupBuyStep3Page.tsx` | 공구 개설 시 수수료 미리보기 |

### 확인 방법
1. `/mypage` — 이름 반짝 효과·등급 배지  
2. `/mypage/tier` — 다음 등급까지 남은 마일리지·수수료 비율  
3. `/open/step3` — 선택 품목 기준 등급별 수수료

## #7 공구 만들 때 가격 입력 제거 · 플랫폼 품목만 선택

### 원문 요청
> 공구만들때 가격입력 빼줘, 우리의 서비스는 쿠팡처럼 우리가 재공하는 물품과 가격이정해져있고 거기서 물품만 정하는거야 … 가격이 이미 저렴해져있고 물품을 고르기만 하면돼

### 완료 여부
✅ **완료** (10차)

### 수행 내용

**플랫폼 카탈로그** (`src/data/productCatalog.ts`) — 5종 품목, `pricePerPerson` 고정.

| ID | 품목 | 공구가/인 |
|----|------|-----------|
| 1 | 3겹 화장지 30롤 | 4,200원 |
| 2 | 액체 세탁세제 3L | 5,800원 |
| 3 | 주방세제 1L 2개 | 3,500원 |
| 4 | 샴푸+린스 세트 | 8,900원 |
| 5 | 물티슈 80매 10팩 | 12,000원 |

**공구 열기 플로우 변경**

- `/open` 1단계: 카탈로그 그리드에서 **품목만 선택** (가격·상품명·카테고리 직접 입력 필드 **전부 제거**)
- `/open/step2`: 선택 품목 카드 (이미지 + 「가격 고정」 표시)
- `/open/step3`: 카탈로그 단가 기준 팀장 수수료·정산 미리보기
- `AppContext` — `selectedCatalogProductId` 상태

### 관련 파일

| 파일 | 역할 |
|------|------|
| `src/data/productCatalog.ts` | 플랫폼 품목·고정가 |
| `src/pages/OpenGroupBuyPage.tsx` | 카탈로그 선택 UI |
| `src/pages/OpenGroupBuyStep2Page.tsx` | 선택 품목 확인 |
| `src/pages/OpenGroupBuyStep3Page.tsx` | 고정가 기준 수수료 |
| `src/context/AppContext.tsx` | 선택 품목 ID 상태 |

### 확인 방법
1. `/open` — 5종 카탈로그만 표시, 가격 입력란 없음  
2. 품목 선택 → 「가격 수정 불가」 안내 확인  
3. `/open/step3` — 선택 품목 단가로 수수료 계산 확인
