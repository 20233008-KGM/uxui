## 2. vision.md 추가업무 반영

| # | 요청 | 상태 |
|---|------|------|
| 1 | 공구 열기 위치 입력 | ✅ 기존 완료 (`/open` 1단계) |
| 2 | 팀장 지인 초대 → 지인 혜택 이벤트 | ✅ 기존 완료 (`/leader/invite`, `/invite/:code`) |
| 3 | **참여·노출 이중 범위 설정 (마이페이지 + 공구 열기)** | ✅ **이번 작업** |

### vision #3 구현 상세

#### 개념
- **참여 범위 (Y)** — 참여자가 “내가 참여할 공구”로 볼 거리
- **노출 범위 (X)** — 팀장이 “내 공구를 보여줄/참여 허용할” 거리

#### 마이페이지 (`/mypage/neighborhood`)
- 두 범위를 분리 UI로 설정
- `localStorage`에 저장되어 새로고침 후에도 유지

#### 공구 열기 (`/open` 1단계, `/open/step3`)
- 1단계: 이번 공구 노출 범위 선택 (마이페이지 기본값 연동)
- 3단계: 선택된 노출 범위 요약 표시

#### 탐색·참여 적용
- 홈·탐색·슬라이드: `참여 범위 ∩ 팀장 노출 범위` 필터
- 상세: 범위 밖이면 **참여 버튼 비활성화** + 경고
- 결제: 범위 밖 접근 시 상세로 리다이렉트

#### 관련 파일
- `src/context/AppContext.tsx` — `participationRange`, `leaderVisibilityRange`, `openVisibilityRange`
- `src/data/locations.ts` — `canJoinGroupBuy()`
- `src/data/groupBuys.ts` — `visibilityRange`, `filterVisibleGroupBuys()`
- `src/pages/NeighborhoodSettingsPage.tsx`

## 3. edge_case.md 대응 (이번 보완)

| 섹션 | 예외 | 대응 |
|------|------|------|
| §1 팀장 | 수수료 부족 → 팀장 역할 기피 | 공구 열기 3단계·포인트 페이지에 **티어별 보너스** (5명 +500P, 8명 +1,500P 등) |
| §3 수령 | 수령 기간 내 미수령 | **`/pickup/overdue/:txId`** 수령 기한 초과 안내 화면 |
| §3 수령 | 팀장 보관 공간 부족 | **`/pickup/storage/:txId`** 대체 수령·분쟁 안내 화면 |
| §3 수령 | 수령 확인 연계 | `/confirm/:txId`에서 D-2 카운트·두 화면 링크 |
| §4 구조 | 콜드스타트 | 홈에서 **범위 내 공구 0개 시 자동 배너** + 지인 초대 CTA |
| §4 구조 | 동네 범위 | 이중 범위 설정 + 탐색/참여 **실제 필터·차단** |
| §4 구조 | 알림 주기 오류 | 중복 방지 토글·알림 주기 **localStorage 영속화** |

### 기존 유지 (이전 작업)
- §1 먹튀/연락두절: `/leader/unavailable`, `/dispute/:txId`
- §1 프라이버시: 동 단위 공개, 상세 주소 확정 후 공개
- §2 무산 환불: `/recruitment/failure` → `/refund/:txId`
- §2 결제 후 취소: `/recruitment/member-cancel`
- §2 불량/허위: 분쟁 유형 `defect`, `mismatch`
- §3 내 것 아님: 분쟁 유형 `wrong_item`
- §5 위치 공유: 공구 열기 1단계 필수 위치 입력

## 5. 데모 확인 방법

```bash
npm run dev
```

1. **이중 범위:** `/mypage/neighborhood` → 참여 500m, 노출 2km 등 설정 → `/explore` 목록 변화 확인  
2. **참여 차단:** `/detail/5` (남현동, 멀리) + 참여 범위 1km → 버튼 비활성  
3. **콜드스타트:** 참여 범위 500m로 좁히면 홈에 콜드스타트 배너  
4. **수령 예외:** `/confirm/tx-001` → 기한 초과·보관 부족 링크  
5. **공구 열기:** `/open` → 위치 + 노출 범위 → step3 요약
